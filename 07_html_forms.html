Form structure (1)

  • Every form, even the smallest, should be wrapped in a <form> element with an action attribute and contain a submit button: 
    <form action="#">
  ...
  <p><input type="submit" value="Send" /></p>
</form>

Form structure (2)

  • The <form> element should also include anything that belongs to the form, also titles, help texts, error messages etc...: 
    <form action="#">
  <h3>Registration form</h3>
  ...
  <p>fields with an * are required</p>
  ...
  <p>
    after submitting the form, you'll receive an email
    with further instructions
  </p>
</form>

Form structure (3)

  • If your form contains more than a few controls, use <fieldset> and <legend> to create groups: 
    <form action="#">
  ...
  <fieldset>
    <legend>Login information</legend>
    ...
  </fieldset>
  <fieldset>
    <legend>Personal information</legend>
    ...
  </fieldset>
  <fieldset>
    <legend>Preferences</legend>
    ...
  </fieldset>
  ...
</form>

Form structure (4)

  • The most flexible markup for form controls is the <dl>, with
    • <label> inside <dt>
    • control inside <dt>
    • radiobutton or checkboxgroup: group label in <dt> and all options nested inside their <label> in <dd>
    <form action="#">
  ...
    <dl>
      <dt><label for="username">Choose a username:</label></dt>
      <dd><input type="text" id="username" name="username" /></dd>

      <dt><label for="password">Choose a password:</label></dt>
      <dd><input type="password" id="password" name="password" /></dd>

      <dt><label for="comments">Comments:</label></dt>
      <dd><textarea id="desc" name="desc" cols="40" rows="5"></textarea></dd>

      <dt>Interests:</dt>
      <dd>
        <label><input type="checkbox" name="sports" /> sports</label>
        <label><input type="checkbox" name="science" /> science</label>
        <label><input type="checkbox" name="edu" /> education</label>
        <label><input type="checkbox" name="news" /> news</label>
      </dd>
    </dl>
  ...
</form>

Form attributes

  • Attributes of <form>:
    • action contains URL to where the form data is sent (used next year)
    • method can be get or post (used next year)
    • enctype should be defined when uploading files (used next year)

FORMS

Controls

Control attributes

  • Controls:
    • all should have a name attribute (used next year)
    • for attribute of <label> should point to the id attribute of the control it belongs to
    • all radiobuttons belonging to the same group should have identical name attributes and different value attributes

Labels

  • Radionbuttons and checkboxes are nested in their <label>; for and id are not needed here: 
    <label>
  <input type="radio" name="gender" value="f" checked /> Female
</label>
<label>
  <input type="radio" name="gender" value="m" /> Male
</label>
  • We could actually put the other controls in their <label> as well: 
    <label>
  Your name:
  <input type="text" name="name" />
</label>
  • This would make future CSS layout difficult, so stick to this code: 
    <dt><label for="name">Your name:</label></dt>
<dd>
  <input type="text" id="name" name="name" />
<dd>

Text controls

  • Text field: 
    <input type="text" name="name" />
  • Text area: 
    <textarea name="box1" cols="20" rows="5"></textarea>
  • Password field: 
    <input type="password" name="password" />
  • Hidden field: 
    <input type="hidden" name="action" value="xyz" />

Radiobuttons & checkboxes

  • Radiobuttons: 
    <input type="radio" name="group1" value="a" />
<input type="radio" name="group1" value="b" />
<input type="radio" name="group1" value="c" />
  • Checkboxes: 
    <input type="checkbox" name="chb1" />
<input type="checkbox" name="chb2" />
<input type="checkbox" name="chb3" />

Lists

  • Dropdown list: 
    <select name="education">
  <option value="-1">Select...</option>
  <option value="0">Primary school</option>
  <option value="1">Secondary school</option>
  <option value="2">Higer education</option>
  <option value="3">Master</option>
</select>
  • With multiple attribute: 
    <select name="education" multiple>
  <option value="0">banana</option>
  <option value="1">apple</option>
  <option value="2">potato</option>
  <option value="3">avocado</option>
</select>
  • not very practical
  • consider checkboxes or Javascript instead

File upload

  • File upload field: 
    <input type="file" name="photo" />
  • The multiple attribute is supported by all modern browsers and IE10+: 
    <input type="file" name="photo" multiple />

Buttons

  • Submit button: 
    <input type="submit" name="btnOk" value="Send" />
  • Alternative syntax (offers more possibilities): 
    <button type="submit">
  <img src="img/07_html_forms/ok.png" alt="save icon"/> Save
</button>
<button type="submit">
  <img src="img/07_html_forms/cancel.png" alt="cancel icon"/> Cancel
</button>
  • Buttons that do nothing also exist: 
    <input type="button" name="btn1" value="Click me" />
  • only useful with Javascript attached, and even then you'd better use a link
If your <button> only contains an image, e.g. 
<button type="submit">
<img src="img/icon_save.png" alt="save"/>
</button>
make sure to provide an alt attribute with the same value as the text on the image. This greatly helps screen readers.

Polyfills

  • Most HTML5 form features can be polyfilled for unsupporting browsers
  • Polyfill example for date input using html5Widgets polyfill: 
    <!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <title>HTML5 date time examples</title>
  <link rel="stylesheet" href="lib/html5Widgets/js/jscalendar-1.0/calendar-win2k-1.css" />
  <!-- Needed for Date/Time Elements -->
  <script src="lib/html5Widgets/js/jscalendar-1.0/calendar.js"></script>
  <script src="lib/html5Widgets/js/jscalendar-1.0/lang/calendar-en.js"></script>
  <script src="lib/html5Widgets/js/jscalendar-1.0/calendar-setup.js"></script>
  <!-- What glues all the above together -->
  <script src="lib/modernizr-1.5.min.js"></script>
  <script src="lib/html5Widgets/js/EventHelpers.js"></script>
  <script src="lib/html5Widgets/js/html5Widgets.js"></script>
</head>

Design guidelines

Proper control use

  • Links vs. buttons:
    • to handle form actions (expand, submit, next step, cancel...): use a HTML button
    • to handle navigation or page actions (add to cart, delete...): use a HTML link
    • you may CSS-style a link as a button to create a call to action, but use it sparingly to keep the user focussed
    • don't use buttons outside forms
    • don't use dropdowns, radiobuttons etc... for navigation; use a group of links instead
  • Radiobuttons vs. checkboxes vs. dropdowns:
    • to select a single option from a short fixed list: use radiobuttons
    • to select zero or more options: use checkboxes
    • to select a single option from a long variable list: use a dropdown
    • don't use the multiple attribute of dropdowns

Proper labeling

  • Control labels:
    • keep your labels short and descriptive
    • align your labels left and end with ':'
  • Button and link texts:
    • use imperative texts for buttons or calls-to-action ('details' → 'view details', 'more info' → 'read more', 'cart' → 'add to cart'...)
    • don't use icons (delete, edit, cart...) without texts
    • don't shout (capitals, exclamation marks) or try to be funny

Proper forms

  • Form structure:
    • any form longer than a few fields should be split up in fiedsets or steps
    • each fieldset or step should have a clear legend or title
    • be clear about input restrictions and formats (required, date formats, minimum or maximum values...)
    • group required fields as much as possible
    • required fields first, optional fields last
  • Form usage:
    • don't show more than one form at a time
    • economize on your controls; keep your forms as short as possible (do you really need all this information?)
    • avoid difficult to hande controls like dropdowns and radiobuttons
    • for registration: consider a short registration form plus a long profile page
    • forget about login/registration for e-commerce; it simply does not work
  • Designing forms is an important art!
