HTTP

Request / response (1)

  • Web pages are usually delivered through HTTP (HyperText Transfer Protocol)
    • user clicks on a link 'index.html'
    • browser sends HTTP request to the server for the page
    • server returns HTTP response with the html code
    • browser scans html code for any linked resource:
      images, css files, external scripts, fonts...
    • browser sends HTTP request for each resource
    • each received resource is inserted in the page

HTTP Request / response (2)

  • Learn by analyzing traffic with a network tool, e.g. for Toledo home (1, 2, 3, 4) and login (1)
  • Provides information about:
    • status codes (200 ok, 304 not modified, 404 not found...)
    • MIME types (text/html, text/css, image/png...)
    • timeline with load times and latencies
    • request and response headers
    • request methods GET and POST
    • cookies
    • ...

Load times

  • Observations:
    • first the html page is fully loaded
    • then other resources are loaded ± in order of appearance
    • number of parallel downloads is limited (usually 2 – 7)
  • Limiting number of resources will reduce page load time! (and improve your Google ranking)
  • → group scripts & css in larger files,
    combine images in CSS sprites...

HTTP headers

  • Request and response are preceded by a header
  • Request header contains
    • info about accepted MIME types, charset, encoding, language...
    • info about the browser and operation system ('User-agent')
    • URL where the request was made ('Referrer')
    • available cookies
  • Response header contains
    • info about the server
    • served content MIME type

GET/POST

  • Two possible request formats: GET and POST
  • GET:
    • default request format
    • extra parameters — e.g. from a <form> — are attached to the request URL; format ...?q=test&lang=nl
    • page can be bookmarked with parameters
    • not safe: parameter values are visible in browser history
  • POST:
    • extra parameters are added to the request body
    • files upload is only possible with POST
    • page cannot be bookmarked with
      parameters
    • little bit safer than GET:
      parameters are more hidden
      (but beware of network sniffers!)

Cookies

  • Name–value pairs
  • Contain information left behind by webpages for autologin, user preferences...
  • Browser limits number and size of cookies
  • User may block or remove cookies at any time
  • Likely to be replaced by more flexible
    localStorage and sessionStorage
    in HTML5

Links in HTML

Examples

  • Example (notice the title attribute): 
    <a href="index.html"><h1>Site Title Here</h1></a>

<ul>
  <li>
    <a href="page1.html" title="edit this item">
      <img src="img/04_html_links/edit.gif" alt="edit icon" />
    </a>
  </li>
  <li>
    <a href="page1.html" title="delete this item">
      <img src="img/04_html_links/del.gif" alt="delete icon" />
    </a>
  </li>
</ul>
  • at older times a target attribute also existed, but today it is disallowed ('deprecated')
A good descriptive title attribute is especially important when linking an image, as screen readers (used by visually impaired) cannot read images.

Relative paths

  • Make all your internal links relative: 
    <!-- file 'index.html' -->
  ...
  <link rel="stylesheet" href="css/styles.css" />
  <link rel="stylesheet" href="https://fonts.googleapis.com/css?family=Lato">
  <script src="js/lib/require/require.js" /></script>
</head>
<body>
  ...
  <p><img src="img/pictures/photo.jpg" alt="my picture" /></p>
  ...
    /* file 'css/styles.css' */
h1 {
  background-image: url(../img/logo.jpg);
  ...
}
  • Relative links start from the current file folder

Meaningful link texts

  • Avoid 'click here', 'click there' etc...:
  • Provide meaningful text for links; compare 
    click <a href="h2.html">here</a> to go to next chapter
  • with 
    go to <a href="h2.html">chapter 2</a>
  • The exact words used in a link pointing to your site are very important for your Google ranking!

Anchors

  • Are links that jump to another location in the same page
  • Link to id="someId" with href="#someId": 
    ...
<h2><a id="top">Jump to a specific chapter:</a></h2>
<ul>
  ...
  <li><a href="#chapter3">chapter 3</a></li>
</ul>
...
<h3><a id="chapter3">Chapter 3</a></h3>
<p>
  Lorem ipsum dolor sit amet, consectetur adipisicing elit...
  <a href="#top">back to top</a>
</p>
...

      alt="Australia" />
   ...
</map>

Link relations

  • Give more meaning to your links with the rel attribute: 
    <a rel="author" href="http://myblog.com/aboutme">Rogier</a>
<a rel="external"
href="http://notmysite.com">tutorial</a>
<a rel="next" href="http://myblog.com/page2.php">
<a rel="search"
href="http://myblog.com/search">search this blog</a>
<a rel="tag"
href="http://myblog.com/category/games">games posts</a>
  • Only introduced in HTML5, and probably not very critical to your website yet
